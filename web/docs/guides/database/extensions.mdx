拡張とは、その名の通り、Postgresのコア部分ではない機能でデータベースを「拡張」するものです。
Supabaseには、最も便利なオープンソースの拡張がいくつかプリインストールされています。


### 拡張の有効化と無効化

defaultValue="UI"

<TabItem value="UI">

```sh

1. データベースのページへ移動
2. サイドバーの「Extensions」をクリック 
3. 有効化/無効化したい拡張を探す 
4. トグルをクリック
```

<TabItem value="SQL">

```sql 

-- 例：「pgtap」拡張を有効化し、インストールされていることを確認
create extension pgtap with schema extensions;

-- 例：「pgtap」拡張を無効化
drop extension pgtap;

```

SQLコードが`create extension`（拡張を作成）していても、これは「拡張を有効にする」と同じことです。
拡張を無効にするには、`drop extension`を呼びだします。

SQLコード、`create extension <extension-name> with schema extensions`を実行した際、パーミッションの問題を引き起こすことがあります。例えば、`dblink`, `http`, `pg_cron`などがあります。

</TabItem>

</Tabs>


### 拡張の全リスト

Supabaseには50以上の拡張があらかじめ設定されています。また、独自のSQL拡張を、SQLエディターを使ってデータベースに直接インストールできます。
